## 1. 项目概述

### 1.1 背景

足迹APP作为面向自由行用户的旅行规划工具，目前的核心能力集中在目的地POI发现、行程笔记和游记分享上。智能行程规划功能当前存在以下不足：

- **缺乏交通闭环**：用户需在多个APP之间切换来规划往返交通
- **信息孤岛**：目的地的时间敏感型信息（活动、节庆、气候）未能与行程规划有机结合
- **决策支持薄弱**：用户自行搜索大量碎片信息，规划效率低

### 1.2 目标

| 维度 | 目标 |
|------|------|
| 用户体验 | 从"出发地→目的地→返回"一站式完成行程规划 |
| 信息密度 | 用户可感知的行程信息附加价值提升 |
| 转换率 | 从行程草稿到最终出行确认的转化率提升 |

### 1.3 成功指标

- 交通规划功能使用率 ≥ 当前活跃用户的 30%
- 活动/气候建议的点击查看率 ≥ 50%
- 用户单次规划完成时间缩短 40%+

## 2. 功能需求详述

### 2.1 来回交通规划

#### 2.1.1 核心能力

**功能描述：** 用户输入出发地、目的地、出行日期后，系统自动查询并推荐往返交通工具方案，包含时间、价格、时长等信息，并支持用户选择后将交通信息嵌入行程。

#### 2.1.2 支持的交通方式

| 交通方式 | 数据源要求 | 展示信息 |
|----------|-----------|---------|
| 飞机 ✈️ | 航班API（携程/去哪儿/航司直连） | 航班号、起飞/到达时间、航站楼、价格、行李额、准点率 |
| 高铁/火车 🚄 | 12306 / 第三方API | 车次、发车/到达站、时间、座位类型及价格、余票数 |
| 长途大巴 🚌 | 客运站数据 | 班次、时间、票价 |
| 自驾 🚗 | 地图导航API（高德/百度） | 路线、里程、预计耗时、过路费、加油站提示 |

#### 2.1.3 交互流程

```
[用户输入]                    [系统处理]                      [输出]
┌─────────────┐          ┌─────────────────┐          ┌──────────────┐
│ 出发地       │          │ 多源数据聚合      │          │ 方案列表      │
│ 目的地       │ ──────→  │ 智能排序（时间/   │ ──────→  │ 方案对比      │
│ 出行日期     │          │ 价格/便利度）     │          │ 选定嵌入行程  │
│ 返程日期     │          │ 实时余位检查      │          │ 自动占位提醒  │
│ 偏好设定     │          │ 个性化推荐       │          │ 价格变动订阅  │
└─────────────┘          └─────────────────┘          └──────────────┘
```

#### 2.1.4 关键功能点

| 功能点 | 详细说明 | 优先级 |
|--------|---------|--------|
| **多方案对比** | 同一路段提供飞机vs高铁vs自驾的多维度对比卡片 | P0 |
| **往返捆绑** | 自动检测去程和回程日期的连带方案，推荐最优往返组合 | P0 |
| **中转推荐** | 无直达时自动规划中转方案（含中转地停留建议） | P1 |
| **智能组合** | 去程飞机+回程高铁等混搭方案推荐 | P1 |
| **价格日历** | 日历视图展示出发日期前后7天/30天的价格波动 | P1 |
| **价格提醒** | 选定方案后支持价格变动推送提醒 | P2 |
| **多人同乘** | 家庭/团队出行时人数维度纳入价格计算 | P2 |
| **订单跳转** | 选定方案后一键跳转至第三方完成支付（或内嵌支付） | P0 |
| **交通+住宿联动** | 到达时间影响住宿check-in安排的智能推荐 | P2 |

### 2.2 时间敏感型智能建议

#### 2.2.1 功能定义

在用户选定旅行时间段后，系统自动挖掘并推送以下三类时间敏感信息，并将其与行程决策深度绑定。

#### 2.2.2 三大信息维度

##### (A) 特别活动 🎉

| 子类型 | 示例 | 数据来源 |
|--------|------|---------|
| 节庆/节日 | 泼水节、圣诞节市集、中秋节灯会 | 目的地旅游局/节庆日历API |
| 音乐会/演出 | 音乐节、演唱会、戏剧季 | 大麦/秀动/本地票务 |
| 体育赛事 | 马拉松、F1、世界杯预选赛 | 赛事日程API |
| 展览/展会 | 艺术展、漫展、科技展 | 本地活动平台 |
| 限定快闪 | Pop-up市集、限定主题店 | 社交媒体爬取/UGC |

##### (B) 季节限定风景 🌸🍁

| 子类型 | 示例 | 触发条件 |
|--------|------|---------|
| 花季 | 樱花、薰衣草、油菜花海 | 日期+地理+物候数据 |
| 红叶季 | 枫叶、银杏 | 日期+地理+气象预测 |
| 雪景/冰雕 | 初雪、雾凇、冰灯节 | 气候数据+历年均值 |
| 动物迁徙 | 鲸鱼迁徙、萤火虫季 | 生物日历+保护区数据 |
| 星空/极光 | 银河季、极光爆发期 | 天文日历+太阳活动预测 |

##### (C) 气候变化与出行建议 🌤️

| 子类型 | 说明 | 建议输出 |
|--------|------|---------|
| 季节气候常态 | 当地该时段的历史气温、降水概率 | 行李准备建议 |
| 极端天气预警 | 台风/暴雨/暴雪/高温等预警 | 调整行程提醒 |
| 空气质量 | 花粉指数、PM2.5、能见度 | 防护建议（口罩/药物） |
| 日出日落 | 日出/日落时间影响景点开放 | 摄影黄金时段推荐 |
| 潮汐/海浪 | 海边活动相关 | 赶海/冲浪适宜时段 |

#### 2.2.3 展示策略

**分层展示原则：**

```
第一层 — 行程概览页
└── 时间敏感摘要卡片（"你的旅程正值🌸樱花季！"）

第二层 — 每日详情页
└── 当日活动推荐 + 天气建议 + 穿搭提醒

第三层 — 弹窗/拓展面板
└── 详细活动列表、购票链接、深度攻略
```

**场景：** 用户计划2026年3月20日-3月27日去日本京都

#### 2.2.5 建议生成优先级规则

```
1. 强关联 + 高价值 → 强制展示（推送通知）
   - 极端天气预警、限定活动、花季/红叶见顷

2. 强关联 + 中价值 → 行程页默认展开
   - 节庆、演出、需预订的热门活动

3. 弱关联 + 高价值 → 折叠/拓展查看
   - 周边小众活动、深层攻略

4. 弱关联 + 低价值 → 不展示
   - 与行程无关的活动信息
```

## 6. 技术实现建议

### 6.1 推荐技术栈

| 层级 | 技术选型 |
|------|---------|
| 前端框架 | Flutter (已有持续投入) / React Native |
| 状态管理 | Provider / Riverpod |
| 后端 | Node.js / Go (高频查询场景) |
| 缓存层 | Redis (缓存交通查询结果, 减少API调用) |
| 推荐引擎 | 规则引擎 + 轻量ML (时间敏感建议排序) |
| 推送服务 | FCM / APNs / 集成Push |

### 6.2 关键性能要求

- 交通方案查询响应时间 ≤ 3s (含多源聚合)
- 时间敏感建议加载 ≤ 1s (缓存命中率 ≥ 90%)
- 天气/活动数据预缓存策略：用户选定行程→后台预加载
- 离线可用：已缓存的方案和建议支持离线查看

### 6.3 数据更新策略

```
[定时任务]                   [事件触发]                [用户主动]
旅游局日历更新  ←————→  气象预警发布  → 推送通知  用户刷新 → 增量更新
产品数据库      ←————→  热点活动上架                下拉刷新
```

### 6.4 冷启动与无数据兜底

- 新目的地：基于历史同季数据生成"典型建议"
- 实时数据不可用：加载上次缓存 + 标注"数据更新于X小时前"
- 无交通数据：提供估算信息 + 跳转第三方查询

## 8. 附录：竞品参考与差异化策略

### 8.1 竞品对标

| 竞品 | 交通规划 | 时间敏感建议 | 我们的差异化机会 |
|------|---------|-------------|----------------|
| **携程/去哪儿** | ⭐⭐⭐⭐⭐ 强 | ⭐⭐ 弱（无季节提醒） | 专注行程视角，非纯预订工具 |
| **穷游行程助手** | ⭐⭐⭐ 中（手动为主） | ⭐⭐⭐ 中（攻略库） | 实时数据+智能推荐 |
| **TripIt** | ⭐⭐⭐ 中 | ⭐ 弱（无本地活动） | 中文市场+本地化 |
| **Google Trips (已停)** | ⭐⭐⭐⭐ 强 | ⭐⭐⭐⭐ 强（生前） | 继承其交互哲学，本土化 |
| **小红书灵感** | ⭐ 无 | ⭐⭐⭐⭐ 强（用户UGC） | 结构化替代碎片化 |

### 8.2 核心差异化策略

1. **一站式决策**：从"查攻略→选交通→看天气→找活动"全链条闭环，无需切换APP
2. **时机推荐引擎**：不只告诉你"那里有什么"，而是告诉你"**现在去那里有什么**"
3. **预见式体验**：在你意识到需要什么之前，系统已经准备好建议（Proactive > Reactive）
4. **轻量化交互**：与足迹APP现有的"干净/文艺"设计风格保持一致，信息呈现克制
5. **数据驱动个性化**：随着用户使用次数增加，推荐越来越精准
